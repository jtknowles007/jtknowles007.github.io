Purpose

Crontab is the program used to install, uninstall, or list the tables used to drive cron.  Each user can have their own crontab.  The user specific files are in /var/spool/cron/crontabs, but they are not intended to be edited directly.  The main system crontab is /etc/crontab.

Prerequisites

  • Superuser credentials

Syntax

Common Usage

crontab -l

View the contents of the current user’s crontab

su
crontab -l

View the contents of the root crontab

su
crontab -u{username} -l

View the contents of specified user’s crontab

crontab -e

Edit crontab of current user

su
crontab -e

Edit root crontab

Superuser Credentials

Placing an entry in the root crontab will allow that entry to be executed with superuser credentials.  Since you cannot execute a cron job that requires superuser credentials from the user’s crontab, it is advisable to place that job in the root crontab.
