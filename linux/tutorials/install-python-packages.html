Install Python Packages

In Debian 12, users are warned not to install packages into the system install of Python.  Per PEP 668, this is to protect against mixing apt provided packages and pip provided packages.  Mixing the two package managers is always a bad idea and the source of many issues.

Prerequisites

  • Python3 installed
  • Pip installed
  • Venv installed

Procedure

Recommended Solutions

  1. The recommended solution to this problem is to use a Virtual Environment to install packages correctly.
  2. The next best solution is to install packages using apt.  The package can be installed as follows
sudo apt install python3

### Alternate Solution - Temporary
If the risks are understood, and you are willing to accept the associated risks, packages can be installed using the following command

```text-x-sh
pip3 install {package} --break-system-packages

Alternate Solution - Permanent

If a more permanent solution to installing packages into the system using pip is desired , the following can be appended to ~/.config/pip/pip.conf

[global]
break-system-packages = true
